# ТЕХНИЧЕСКОЕ ЗАДАНИЕ

на разработку информационной системы

## «LESogram — многопользовательский чат в реальном времени»

------

# 1. Общие положения

## 1.1 Полное наименование системы

Информационная система «LESogram — многопользовательский чат в реальном времени»

## 1.2 Назначение системы

Система предназначена для обмена текстовыми сообщениями между пользователями в режиме реального времени в рамках тематических чат-комнат с использованием WebSocket-соединения.

## 1.3 Цели создания системы

Основными целями создания Системы являются:

1. **Организация мгновенного обмена сообщениями** между пользователями.
2. **Реализация системы чат-комнат** для тематического общения.
3. **Обеспечение безопасной аутентификации пользователей** с использованием JWT.
4. **Сохранение истории сообщений** для последующего просмотра.
5. **Создание базовой архитектуры для расширения** (медиа, уведомления, масштабирование).

------

# 2. Область применения и границы проекта

## 2.1 Область применения

Система предназначена для использования:

- **Конечными пользователями** — для общения в чатах
- **Разработчиками/администраторами** — для развертывания и поддержки системы

## 2.2 Автоматизируемые процессы

| Код   | Процесс                  | Описание                          |
| ----- | ------------------------ | --------------------------------- |
| CP-01 | Регистрация пользователя | Создание учетной записи           |
| CP-02 | Авторизация              | Вход в систему и получение токена |
| CP-03 | Подключение к комнате    | Вход в чат-комнату                |
| CP-04 | Обмен сообщениями        | Отправка и получение сообщений    |
| CP-05 | Получение истории        | Загрузка предыдущих сообщений     |

## 2.3 Исключённые функции

В рамках текущего проекта **не реализуются**:

- Голосовые/видеозвонки
- Push-уведомления
- Масштабируемая кластеризация WebSocket
- Мобильные приложения

------

# 3. Функциональные требования

## 3.1 Общая структура подсистем

Система включает:

- **Подсистема «Клиентская часть» (Frontend)**
- **Подсистема «Серверная часть» (Backend)**
- **Подсистема «Передача данных (WebSocket)»**

------

## 3.2 Подсистема «Аутентификация пользователей»

| Атрибут  | Требование                                                   |
| -------- | ------------------------------------------------------------ |
| UC-01.01 | Пользователь должен иметь возможность зарегистрироваться     |
| UC-01.02 | Ввод логина и пароля обязателен                              |
| UC-01.03 | Пароль должен сохраняться в виде SHA-256 хеша                |
| UC-01.04 | При успешной авторизации выдается JWT-токен                  |
| UC-01.05 | Доступ к WebSocket возможен только при наличии валидного токена |

------

## 3.3 Подсистема «Чат-комнаты»

| Атрибут  | Требование                                             |
| -------- | ------------------------------------------------------ |
| UC-02.01 | Пользователь может подключаться к существующей комнате |
| UC-02.02 | При отсутствии комнаты она создается автоматически     |
| UC-02.03 | Пользователь получает сообщения только своей комнаты   |
| UC-02.04 | Поддерживается несколько пользователей в одной комнате |

------

## 3.4 Подсистема «Обмен сообщениями»

| Атрибут  | Требование                                      |
| -------- | ----------------------------------------------- |
| UC-03.01 | Сообщения передаются в реальном времени         |
| UC-03.02 | Сообщение получают все участники комнаты        |
| UC-03.03 | Поддерживаются системные сообщения (вход/выход) |
| UC-03.04 | Сообщения содержат текст, автора и timestamp    |

------

## 3.5 Подсистема «История сообщений»

| Атрибут  | Требование                                         |
| -------- | -------------------------------------------------- |
| UC-04.01 | При подключении загружаются последние 50 сообщений |
| UC-04.02 | Сообщения отображаются в хронологическом порядке   |
| UC-04.03 | История хранится в базе данных                     |

------

# 4. Нефункциональные требования

## 4.1 Производительность

| Показатель                  | Требование         |
| --------------------------- | ------------------ |
| Задержка доставки сообщений | ≤ 200 мс           |
| Количество подключений      | ≥ 50 пользователей |
| Время ответа API            | ≤ 300 мс           |

------

## 4.2 Надёжность

| Показатель                   | Требование                |
| ---------------------------- | ------------------------- |
| Обработка разрыва соединения | Обязательна               |
| Некорректные подключения     | Автоматическое отключение |
| Потеря данных                | Недопустима               |

------

## 4.3 Безопасность

| Аспект             | Требование                            |
| ------------------ | ------------------------------------- |
| Хранение паролей   | SHA-256                               |
| Аутентификация     | JWT                                   |
| Доступ к WebSocket | Только с токеном                      |
| Защита данных      | Базовая (валидация, проверка токенов) |

------
