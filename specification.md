# Техническое задание  
## Проект: LESogram (многопользовательский чат в реальном времени)

---

# 1. Общая информация

## 1.1 Назначение системы
Разработка веб-приложения для обмена текстовыми сообщениями в реальном времени между пользователями в рамках тематических комнат (chat rooms) с использованием WebSocket-соединения.

## 1.2 Цель проекта
Создать простой аналог мессенджера с поддержкой:
- авторизации пользователей
- комнат (чатов)
- мгновенной доставки сообщений
- сохранения истории сообщений

---

# 2. Функциональные требования

## 2.1 Регистрация пользователя
Система должна позволять:
- создание нового пользователя
- ввод логина и пароля
- сохранение пароля в хешированном виде (SHA-256)

---

## 2.2 Авторизация
Система должна:
- проверять логин и пароль
- выдавать JWT-токен при успешном входе
- ограничивать доступ к WebSocket без токена

---

## 2.3 Чат-комнаты
Пользователь должен иметь возможность:
- подключаться к существующей комнате
- создавать новую комнату при первом входе
- получать сообщения только из своей комнаты

---

## 2.4 Обмен сообщениями
Система должна обеспечивать:
- отправку сообщений в реальном времени
- получение сообщений всеми участниками комнаты
- отображение системных сообщений (вход/выход пользователей)

---

## 2.5 История сообщений
При подключении пользователь должен:
- получать последние 50 сообщений из выбранной комнаты
- видеть сообщения в хронологическом порядке

---

# 3. Нефункциональные требования

## 3.1 Производительность
- доставка сообщений ≤ 200 мс при локальном запуске
- поддержка минимум 50 одновременных подключений (учебный уровень)

## 3.2 Безопасность
- пароли хранятся в виде SHA-256 хеша
- доступ к WebSocket осуществляется только при наличии JWT-токена

## 3.3 Надёжность
- автоматическое отключение некорректных соединений
- обработка разрыва WebSocket-соединения

---

# 4. Архитектура системы

## 4.1 Клиентская часть (Frontend)
- HTML
- CSS (тёмная тема UI)
- JavaScript
- WebSocket API

Функции:
- авторизация
- отправка/получение сообщений
- отображение UI чата

---

## 4.2 Серверная часть (Backend)
- Python
- FastAPI
- WebSocket

Функции:
- обработка WebSocket соединений
- JWT авторизация
- работа с пользователями
- сохранение сообщений

---

## 4.3 База данных
- SQLite

Таблицы:
- users (id, username, hashed_password)
- messages (id, username, room, text, timestamp)

---

# 5. Протокол взаимодействия

## 5.1 HTTP API

### Регистрация

POST /register

### Логин

POST /login

### История сообщений

GET /messages/{room}

---

## 5.2 WebSocket

ws://host/ws/{room}/{username}?token=JWT

Функции:
- отправка сообщений
- получение сообщений в реальном времени
